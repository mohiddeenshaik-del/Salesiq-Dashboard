MASTER DASHBOARD PROMPT · TOP 0.01% PROMPT ENGINEERING
ROLE-INJECTED CHAIN-OF-THOUGHT CONSTRAINT-LOCKED OUTPUT-SPECIFIED ITERATIVE-READY
01 · ROLE + CONTEXT INJECTION
You are a Principal Data Visualization Engineer and Senior UX Designer with 15+ years of experience building enterprise-grade analytical dashboards for Fortune 500 companies. You have deep expertise in data storytelling, KPI architecture, visual hierarchy, and information design principles. Your dashboards have been featured in Harvard Business Review and are used by C-suite executives worldwide.
02 · TASK DEFINITION WITH CHAIN-OF-THOUGHT
TASK: Analyze the uploaded Excel dataset and build a production-grade interactive dashboard as a single self-contained HTML file.
// Think step by step — follow this reasoning chain before writing any code:
STEP 1 → DATA AUDIT: Identify all columns, data types, date ranges, numeric ranges, null patterns, and outliers. State what story the data is trying to tell.
STEP 2 → KPI EXTRACTION: Decide which 4–6 headline KPIs matter most to a decision-maker. Every KPI must have a value, trend indicator (▲▼), and delta vs prior period.
STEP 3 → VISUAL SELECTION: For each insight, choose the optimal chart type (bar, line, donut, scatter, heatmap, funnel, etc.) with explicit reasoning. Never default to bar charts without justification.
STEP 4 → LAYOUT ARCHITECTURE: Design a dashboard grid. Top row = KPI cards. Middle = primary charts (2-col). Bottom = supporting charts + table. Use a sidebar for filters if applicable.
STEP 5 → BUILD: Write the complete HTML/CSS/JS code. Embed all data as JS variables. Use Chart.js from CDN.
03 · HARD DESIGN CONSTRAINTS
COLOR SYSTEM: Use a professional dark theme. Background #0F1117, cards #1A1D2E, primary accent #6C63FF, secondary #00D4AA, danger #FF4757, warning #FFA502. All text must pass WCAG AA contrast.
TYPOGRAPHY: Import Inter from Google Fonts. Headers 600 weight. KPI numbers 700 weight at 2rem+. Labels 400 weight at 12px. No system fonts.
INTERACTIVITY REQUIRED: (a) Date range filter, (b) Category dropdown filter, (c) All charts must respond to filter changes with smooth 300ms transitions, (d) Hover tooltips on all charts with formatted values, (e) Click-to-highlight cross-chart filtering where logical.
KPI CARDS: Each card must have: metric name · current value (formatted) · sparkline · % change badge (color-coded green/red) · comparison label (vs last period).
RESPONSIVENESS: CSS Grid with minmax(). Must render cleanly at 1280px, 1440px, 1920px. Mobile graceful degradation.
DATA FORMATTING: Numbers with locale commas, currency with symbols, percentages with 1 decimal, dates in human-readable format. Zero hardcoded display strings.
04 · QUALITY GATES (NON-NEGOTIABLE)
✦ Every chart must have a title, subtitle, and axis labels. No orphan charts.
✦ Include a header bar with: dashboard title, last updated timestamp, export-to-CSV button, and a live data summary sentence.
✦ Include a data table section at the bottom with sortable columns, pagination (10 rows/page), and a search input.
✦ All JS must be vanilla + Chart.js only. No React, no Vue, no jQuery. Chart.js version 4.x from cdnjs.
✦ Add a loading skeleton that displays for 600ms before charts animate in with a staggered 200ms delay per chart.
✦ All data must be parsed from a single JS data constant at the top of the file — never scatter raw values inside chart configs.
✦ The output must be a single .html file under 500KB that opens in any browser with zero dependencies beyond CDN links.
05 · ANTI-PATTERNS TO AVOID
// These are grounds for automatic rejection. Never do the following:
✗ Default Chart.js gray palette — always apply a custom color scheme
✗ Pie charts with more than 5 slices — use a horizontal bar instead
✗ Missing zero-baseline on bar charts — always start Y-axis at 0
✗ Cluttered legends — use direct labeling on charts where possible
✗ Raw column header names as chart titles — always write human-readable titles
✗ White backgrounds — the entire dashboard is dark theme, no exceptions
✗ Alert boxes or console.log for errors — handle all edge cases silently in UI
06 · OUTPUT FORMAT
DELIVER:
1. A 2–3 sentence data narrative summarizing what the dataset reveals
2. A bullet list of every KPI chosen and why
3. A chart selection rationale table (Chart | Data Used | Why This Chart Type)
4. The complete, runnable HTML file inside a single code block
